# Matrix Sarvam UCS Call Log Settings

# PURPOSE

This guide explains:

* how to sync latest call logs
* where to change IP settings
* what settings should remain unchanged
* how to switch between development and production environments safely

---

# CURRENT SYSTEM ARCHITECTURE

## PBX

Matrix Sarvam UCS PBX

Example PBX IP:
192.168.0.81

(Static IP — should remain fixed)

## DEVELOPMENT MACHINE

Purpose:

* testing
* debugging
* local verification

IP Address:
192.168.0.169

---

## TOWER SERVER (PRODUCTION)

Purpose:

* final deployment
* production usage
* live CRM sync

IP Address:
192.168.0.205

The PBX sends call logs using:

# ACTUAL SMDR SECTIONS THAT REQUIRE IP CHANGE

The following sections inside Jeeves are the ONLY areas where Destination IP changes are normally required for call log syncing.

---

# 1. SMDR POSTING

Inside Jeeves:

Reports
↓
SMDR
↓
SMDR Posting

Current Working Example:

| Setting                | Value         |
| ---------------------- | ------------- |
| Protocol               | Matrix        |
| Destination Port       | Ethernet      |
| Destination IP Address | 192.168.0.169 |
| Port                   | 05001         |
| Process                | Start         |

Purpose:

* sends latest call logs automatically to UniComm backend
* powers real-time CRM sync
* used for latest/live call entries

IMPORTANT:
Only change:

Destination IP Address

when switching between:

* development machine
* production/tower server

---

---

# 3. SMDR REPORT

Inside Jeeves:

Reports
↓
SMDR
↓
SMDR Report

Current Working Example:

Outgoing Call Report:

* Destination IP = 192.168.0.169
* Port = 05001

Incoming Call Report:

* Destination IP = 192.168.0.169
* Port = 05001

Internal Call Report:

* Destination IP = 192.168.0.169
* Port = 05001

Purpose:

* historical call reports
* exported call logs
* old call retrieval
* report synchronization

Critical UI state:

If the report page shows a button like:

Send to Destination Port

then report pushing is not currently being sent/triggered for that report.
This is the Matrix-side setting/state that can make the PBX show fresh call
records internally while UniComm CRM stays stale.

For missing historical/day call logs, check these pages first:

* OG Call Report
* IC Call Report
* Internal Call Report

Each required report must be sent/enabled to the destination port:

192.168.0.205 : 5001

The tower server IP is correct for production. The failure is when the report
delivery/action itself is off or not triggered, even though the destination IP
and port are correct.

IMPORTANT:
Only change:

Destination IP Address

for all report sections when switching between development and production.

# DEVELOPMENT VS PRODUCTION IP SWITCHING

## During Development

Use everywhere:

192.168.0.169

---

## During Production

Change everywhere to:

192.168.0.205

---

# IMPORTANT RULE

When switching environments:

Change ALL these sections together:

* SMDR Posting
* SMDR Online
* SMDR Report

Otherwise:

* partial logs may occur
* live calls may mismatch
* reports may go missing
* dashboard may show incomplete data

# IMPORTANT RULE

ONLY ONE DESTINATION SHOULD BE ACTIVE.

Do NOT send logs simultaneously to:

* 192.168.0.169
* 192.168.0.205

unless a custom relay/replication system exists.

Otherwise:

* duplicate logs may happen
* missing entries may happen
* database conflicts may happen

# SMDR STORAGE SETTINGS

Inside Jeeves:

Reports
↓
SMDR
↓
SMDR Storage

Recommended:

| Setting     | Recommended Value |
| ----------- | ----------------- |
| Storage     | ENABLED           |
| Retention   | MAXIMUM           |
| Auto Delete | DISABLED          |

Reason:

* preserves historical call logs
* avoids accidental deletion
* maintains long-term call history

---

# FIREWALL SETTINGS

Normally no changes required.

But ensure:

* backend listener port is allowed
* Node.js/backend app is allowed
* LAN communication is not blocked

---

# NETWORK SETTINGS

Inside Jeeves:

Basic Settings
↓
Network Parameters

Recommended:

| Setting | Recommended |
| ------- | ----------- |
| IP Type | Static      |
| PBX IP  | Fixed       |
| Gateway | Router IP   |
| DNS     | Stable DNS  |

Reason:

* prevents PBX disconnection
* prevents CRM sync failures
* prevents SMDR posting failures

---

# RECOMMENDED ITEMS TO BACKUP

Backup:

* call logs database
* SMDR raw logs
* extension mapping
* recording metadata
* recording file paths

---

# THINGS THAT SHOULD NEVER BE DONE

NEVER:

* remove PBX storage during operation
* disable SMDR Online
* disable SMDR Posting
* enable auto-delete casually
* change PBX static IP frequently
* reboot router during active calls
* use both destination servers simultaneously without replication logic
