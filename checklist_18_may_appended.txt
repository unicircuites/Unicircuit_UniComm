# PBX Call Recordings — Duplicate Fix

## Problem Identified

### Symptoms
- Local system showing:
  - ~766 pages
  - ~15320 recordings

- Tower server showing:
  - ~383 pages
  - ~7666 recordings

### Root Cause

PBX_RECORDINGS_DIR pointed to:

PBX_RECORDINGS_DIR=C:\MatrixVMS\Voicemail_Backup

Matrix SARVAM UCS automatically creates timestamp-based backup folders:

16May_2026_175604
16May_2026_183002

Application recursively scanned ALL folders.

Same .wav recordings existed in multiple backup snapshots.

This caused duplicate recordings in UI.

# Permanent Deduplication Fix Applied

## Previous Logic

Old code directly used:

const files = getAllRecordingFiles(REC_DIR)

This loaded duplicate recordings from multiple backup folders.

---

## New Logic Implemented

### Added Deduplication by Filename

Implemented:

const allFiles = getAllRecordingFiles(REC_DIR);

const uniqueMap = new Map();

for (const fullPath of allFiles) {
  const filename = path.basename(fullPath);
  const stat = fs.statSync(fullPath);

  if (
    !uniqueMap.has(filename) ||
    stat.mtime > uniqueMap.get(filename).mtime
  ) {
    uniqueMap.set(filename, {
      fullPath,
      mtime: stat.mtime
    });
  }
}

### Result
- scans all UCS backup folders
- keeps only unique filenames
- preserves latest modified copy
- prevents duplicate UI records
- future-safe for unlimited backup folders
