  KEY DEBUG MARKERS
═══════════════════════════════════════════════════════════════════════════════

✅ SUCCESS
   Indicates successful operation or correct configuration

❌ ERROR
   Indicates failure or incorrect configuration

⚠️ WARNING
   Indicates unusual condition that may cause issues

💓 HEARTBEAT
   Periodic status message showing service is alive

📋 STEP MARKER
   Indicates current phase of process

📥 DATA RECEIVED
   Indicates incoming data from PBX

📤 DATA SENT
   Indicates outgoing data to PBX

🤝 HANDSHAKE
   Indicates handshake protocol event

⏱️ TIMEOUT
   Indicates timeout event

═══════════════════════════════════════════════════════════════════════════════
  CRITICAL DEBUG POINTS
═══════════════════════════════════════════════════════════════════════════════

1. INITIALIZATION
   Look for: PBX_HOST is valid: 192.168.0.81 ✅
   If wrong: Check .env file, PBX_HOST must be PBX IP, not server IP

2. SERVER STARTUP
   Look for: ✅ Listening on 0.0.0.0:5001
   If wrong: Check port not in use, firewall allows port

3. INBOUND CONNECTION
   Look for: Is PBX? = ✅ YES — matches PBX_HOST
   If wrong: Check PBX SMDR Posting Destination IP configuration

4. HANDSHAKE
   Look for: ✅ ENQ (0x00) DETECTED — Handshake initiated!
   If wrong: PBX SMDR service not fully started, restart it

5. DATA RECEIVED
   Look for: Bytes received = [number > 0]
   If wrong: PBX not sending data, make test call

6. RECORD PARSING
   Look for: ✅ Parsed successfully
   If wrong: SMDR record format not recognized

═══════════════════════════════════════════════════════════════════════════════
  TROUBLESHOOTING QUICK REFERENCE
═══════════════════════════════════════════════════════════════════════════════

SYMPTOM: Server starts but no connection from PBX
ACTION:
  1. Check: [SMDR-DEBUG] PBX_HOST is valid: 192.168.0.81 ✅
  2. Check: [SMDR-DEBUG] ✅ Listening on 0.0.0.0:5001
  3. If no "INBOUND CONNECTION RECEIVED" message:
     - PBX not sending to this server
     - Check PBX SMDR Posting Destination IP
     - Click "Start" button on PBX

SYMPTOM: Connection received but no handshake
ACTION:
  1. Check: [SMDR-DEBUG] Is PBX? = ✅ YES
  2. Check: [SMDR-DEBUG] ⏱️  HANDSHAKE TIMEOUT
  3. If timeout:
     - Restart SMDR service on PBX
     - Verify SMDR Online is enabled

SYMPTOM: Handshake complete but no data
ACTION:
  1. Check: [SMDR-DEBUG] ║ DATA RECEIVED
  2. If no data events:
     - Make test call on PBX
     - Check PBX SMDR Online is enabled

SYMPTOM: Data received but not parsed
ACTION:
  1. Check: [SMDR-DEBUG] ❌ Failed to parse record
  2. Check: [SMDR-DEBUG] ❌ No valid format detected
  3. Review raw line content in logs
  4. May need to adjust parsing logic
