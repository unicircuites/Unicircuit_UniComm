1. [✓] Investigated missing May 14 call logs
   - User checked Matrix/PBX call records for May 14, 2026.
   - Dashboard filter for 05/14/2026 showed "No call records found."
   - Database check confirmed call_logs had no rows stored under call_date = 2026-05-14 before repair.

2. [✓] Root cause found: SMDR fixed-width date parsing was brittle
   - File: backend/services/matrixSmdr.js
   - Raw PBX lines contained correct dates such as:
     * 14-05-26
     * 13-05-26
   - Parser was reading date/time by hardcoded column offsets.
   - In some Matrix SMDR line formats this shaved or shifted date digits.
   - Example issue:
     * raw 14-05-26 was stored as 2026-05-04
     * raw 13-05-26 was stored as 2026-05-03 or other wrong dates

3. [✓] Future SMDR date parsing fixed
   - File: backend/services/matrixSmdr.js
   - Fixed-width parser now first extracts the real date/time using a pattern:
     * DD-MM-YY HH:MM:SS
     * D-MM-YY HH:MM:SS
     * DD-MM-YYYY HH:MM:SS
   - Hardcoded fixed-width offsets remain only as fallback.
   - This prevents future live PBX rows from being stored under the wrong date.

4. [✓] Existing wrong call_dates repaired from raw_line
   - Table: call_logs
   - Repaired rows by extracting date/time from raw_line.
   - Updated 246 rows whose stored call_date/call_time did not match the raw SMDR date/time.
   - After repair, 2026-05-14 has 49 call records.

5. [✓] PBX Sync Latest wording corrected
